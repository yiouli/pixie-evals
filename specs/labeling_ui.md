# Spec: Custom Labeling UI Component System

## Overview & Design Philosophy

Users define labeling UI components as `.tsx` files in a designated folder. The framework:
1. **Discovers** all `.tsx` files in that folder automatically — no registration calls needed
2. **Scaffolds** new component files via a CLI command (generation logic TBD; this spec covers the output contract only)
3. **Bundles** each file with esbuild lazily when `serve()` is called
4. **Serves** each component as a standalone labeling page at `/labeling/<filename>?id=<input_object_id>`

The filename is the component's identity. The exported `InputProps` interface is the data contract.

---

## Folder Convention

The user places their labeling components in a folder at the project root:

```
my_project/
  labeling/                     ← the components folder (configurable, default: "labeling")
    trace_comparison.tsx
    llm_output_review.tsx
    classification_check.tsx
  main.py
```

No glob pattern, no config file. Every `.tsx` file directly inside this folder (non-recursive) is treated as a labeling component.

---

## TSX File Contract

Each `.tsx` file must conform to this structure, which the scaffold command generates:

```tsx
// labeling/trace_comparison.tsx
// Generated by: yourtool scaffold trace_comparison
// Rename this file to rename the labeling route.
// The route will be served at: /labeling/trace_comparison?id=<input_object_id>

// The props interface IS the data contract between this component and the framework.
// The framework passes the input object loaded by ID directly as these props.
// Field names must match the keys in the stored input object.
export interface InputProps {
  input: string;
  output: string;
  expected: string;
  score: number;
  metadata: Record<string, unknown>;
}

// Default export is required. Component name must match the filename (PascalCase).
export default function TraceComparison(props: InputProps) {
  const { input, output, expected, score } = props;

  return (
    <div>
      {/* Your labeling UI here */}
    </div>
  );
}
```

### Rules the Scaffold Enforces (and the Framework Validates)

| Rule | Detail |
|------|--------|
| Default export required | Must be a React component (function or class) |
| `InputProps` interface must be exported | Framework uses it for documentation; runtime does not enforce shape |
| File name = route segment | `trace_comparison.tsx` → `/labeling/trace_comparison` |
| No sub-folders | Only files directly in the `labeling/` dir are picked up |
| `.tsx` only | `.jsx`, `.ts`, `.js` are ignored (users write typed components) |

### What the Framework Does NOT Validate at Runtime
- That the input object's shape matches `InputProps` — this is a TypeScript dev-time concern
- That the component renders without errors — runtime errors are caught by an error boundary in the shell

## 1. `_components/_registry.py`

### Purpose
Central in-memory store mapping component names to their source and bundle paths.

### Data Model

```python
from dataclasses import dataclass
from pathlib import Path

@dataclass
class RegisteredComponent:
    slot: str           # e.g. "trace_comparison"
    src_path: Path      # original user .tsx file, resolved absolute
    bundle_path: Path   # transpiled ESM output path in tmpdir
```

### Public Interface

```python
_registry: dict[str, RegisteredComponent] = {}

def get_component(slot: str) -> RegisteredComponent | None: ...
def set_component(slot: str, component: RegisteredComponent) -> None: ...
def list_slots() -> list[str]: ...
```

No persistence needed. The registry is rebuilt each process run.

## 2. `_components/_esbuild.py`

### Purpose
Locate, download, cache, and return the path to an esbuild binary for the current platform. Must work with no Node, no npm, and no internet after first run.

### Constants

```python
ESBUILD_VERSION = "0.20.2"

# Maps (platform.system().lower(), platform.machine().lower()) → (os_slug, arch_slug)
# Slugs match the @esbuild npm package naming convention
PLATFORM_MAP: dict[tuple[str, str], tuple[str, str]] = {
    ("linux",   "x86_64"):  ("linux",  "x64"),
    ("linux",   "aarch64"): ("linux",  "arm64"),
    ("linux",   "armv7l"):  ("linux",  "arm"),
    ("darwin",  "x86_64"):  ("darwin", "x64"),
    ("darwin",  "arm64"):   ("darwin", "arm64"),
    ("windows", "amd64"):   ("win32",  "x64"),
    ("windows", "arm64"):   ("win32",  "arm64"),
}
```

### Cache Location

```
~/.cache/yourtool/esbuild/<ESBUILD_VERSION>/<os_slug>-<arch_slug>/esbuild[.exe]
```

Use `platformdirs` if available, fall back to `Path.home() / ".cache"`. Do NOT use a temp directory — the binary must persist across runs.

### Environment Override

Before any download logic, check:
```python
env_path = os.environ.get("YOURTOOL_ESBUILD_PATH")
if env_path:
    p = Path(env_path)
    if not p.exists():
        raise FileNotFoundError(f"YOURTOOL_ESBUILD_PATH set but not found: {p}")
    return p
```

### Download Logic

Download the platform-specific esbuild binary from the npm registry tarball using `urllib.request` only (stdlib). Do NOT use `npm` or `npx`.

URL pattern:
```
https://registry.npmjs.org/@esbuild/{os_slug}-{arch_slug}/-/{os_slug}-{arch_slug}-{ESBUILD_VERSION}.tgz
```

Steps:
1. Fetch the URL with `urllib.request.urlopen`
2. Decompress with `tarfile` (mode `"r:gz"`) from an `io.BytesIO` buffer
3. Extract the member at path `package/bin/esbuild` (or `package/esbuild.exe` on Windows)
4. Write bytes to the cache path
5. `chmod` to add execute bit on non-Windows: `dest.stat().st_mode | stat.S_IEXEC | stat.S_IXGRP | stat.S_IXOTH`
6. Verify the binary runs: `subprocess.run([str(path), "--version"], check=True)`

### Error Handling

- Unsupported platform → raise `RuntimeError` with clear message listing `YOURTOOL_ESBUILD_PATH` as the escape hatch
- Network failure → re-raise with: `"Failed to download esbuild. Check your internet connection or set YOURTOOL_ESBUILD_PATH."`
- Corrupt download → caught by the `--version` verification step

### Public Interface

```python
def get_esbuild_binary() -> Path:
    """
    Returns path to esbuild binary for the current platform.
    Downloads and caches on first call. Subsequent calls return immediately.
    """
```

Use a module-level `_cached_path: Path | None = None` to skip disk checks after first successful resolution within a process.

## 3. `_components/_bundler.py`

### Purpose
Invoke esbuild to transpile a user `.tsx` file into a single ESM bundle.

### Output Directory

```python
import tempfile
_bundle_dir = Path(tempfile.mkdtemp(prefix="yourtool-components-"))
```

Created once at module import time. Ephemeral per process — bundles are regenerated on each `serve()`.

### esbuild Invocation

```python
def bundle_component(src: Path, slot: str) -> Path:
    esbuild = get_esbuild_binary()
    out = _bundle_dir / f"{slot}.js"

    cmd = [
        str(esbuild),
        str(src),
        "--bundle",
        "--format=esm",
        f"--outfile={out}",
        "--jsx=automatic",             # no need for user to import React
        "--external:react",
        "--external:react-dom",
        "--external:react/jsx-runtime",
        f"--loader:{src.suffix}={src.suffix.lstrip('.')}",
        "--minify",
        "--sourcemap=inline",
        "--log-level=error",
    ]

    result = subprocess.run(cmd, capture_output=True, text=True)

    if result.returncode != 0:
        raise ComponentBundleError(
            f"Failed to bundle component for slot '{slot}':\n{result.stderr}"
        )

    return out
```

### Custom Exception

```python
class ComponentBundleError(Exception):
    """Raised when esbuild fails to transpile a user component."""
```

The full esbuild stderr is included in the message so the user sees exactly which line of their component failed.

## 4. `_components/_scanner.py`

### Purpose
Walk the components folder, trigger bundling for each `.tsx` file, and populate the registry.

### Public Interface

```python
def scan_and_register(components_dir: Path) -> list[str]:
    """
    Discover all .tsx files in components_dir (non-recursive).
    Bundle each one and register in the registry.
    Returns list of successfully registered component names.
    Prints a warning for any file that fails to bundle — does not raise.
    """
```

### Implementation

```python
def scan_and_register(components_dir: Path) -> list[str]:
    if not components_dir.exists():
        print(f"[yourtool] Components folder not found: {components_dir}. No labeling components loaded.")
        return []

    tsx_files = sorted(components_dir.glob("*.tsx"))  # sorted for deterministic ordering

    if not tsx_files:
        print(f"[yourtool] No .tsx files found in {components_dir}.")
        return []

    registered = []
    for tsx_file in tsx_files:
        slot = tsx_file.stem  # "trace_comparison.tsx" → "trace_comparison"
        try:
            bundle_path = bundle_component(tsx_file, slot)
            set_component(slot, RegisteredComponent(
                slot=slot,
                src_path=tsx_file,
                bundle_path=bundle_path,
            ))
            registered.append(slot)
            print(f"[yourtool] ✓ {slot}")
        except ComponentBundleError as e:
            print(f"[yourtool] ✗ {slot} (bundle failed)")
            print(str(e))

    print(f"[yourtool] {len(registered)}/{len(tsx_files)} components ready.")
    return registered
```

### When Is This Called?

`scan_and_register()` is called **once, inside `serve()`**, before the server starts accepting connections. It is NOT called at import time — nothing happens until the user explicitly starts the server.

```python
from pathlib import Path

_components_dir: Path = Path("labeling")  # default, relative to cwd at serve() time

def set_components_dir(path: str | Path) -> None:
    """
    Override the default components folder.

        yourtool.set_components_dir("./my_ui_components")

    _components_dir = Path(path)
```

The `serve()` function (defined in the main `yourtool` package) resolves and scans the components dir:

```python
def serve(port: int = 8080) -> None:
    from ._components._scanner import scan_and_register

    resolved_dir = _components_dir if _components_dir.is_absolute() \
                   else Path.cwd() / _components_dir

    print(f"[yourtool] Scanning {resolved_dir} for labeling components...")
    scan_and_register(resolved_dir)

    # ... rest of server startup
```

## 6. `_components/_server.py` — HTTP Routes

All routes are added to the existing FastAPI app via an `APIRouter`. Mount it in the main app file:

```python
from yourtool._components._server import router as components_router
app.include_router(components_router)
```

### Component Bundle Routes

```python
@router.get("/api/components/{slot}.js")
async def serve_component(slot: str):
    component = get_component(slot)
    if component is None:
        raise HTTPException(status_code=404, detail=f"No component registered for slot '{slot}'")
    return FileResponse(
        component.bundle_path,
        media_type="application/javascript",
        headers={"Cache-Control": "no-cache"},
    )

@router.get("/api/components")
async def list_components():
    return {"slots": list_slots()}
```

### Labeling Page Route

```python
@router.get("/labeling/{component_name}")
async def labeling_page(component_name: str, id: str):
    component = get_component(component_name)
    if component is None:
        raise HTTPException(
            status_code=404,
            detail=f"No component registered for '{component_name}'. Available: {list_slots()}"
        )
    return HTMLResponse(content=render_shell(component_name=component_name, input_id=id))
```

### Input Object Stub Route

```python
@router.get("/api/inputs/{input_id}")
async def get_input(input_id: str):
```

## 7. `_components/_shell.py` — HTML Shell Generator

### Purpose
Produce a self-contained HTML page per labeling component that:
1. Sets up React via an import map pointing to vendored ESM files
2. Dynamically imports the component bundle from `/api/components/{component_name}.js`
3. Fetches the input object from `/api/inputs/{id}`
4. Renders the component with the fetched data spread as props
5. Shows a loading state and an in-page error for each failure mode

### `render_shell()` Function

```python
def render_shell(component_name: str, input_id: str) -> str:
    return SHELL_TEMPLATE.format(
        component_name=component_name,
        input_id=input_id,
        title=component_name.replace("_", " ").title(),
    )
```

### `SHELL_TEMPLATE`

A Python string constant — no file read, no asset pipeline:

```python
SHELL_TEMPLATE = """<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>{title}</title>
  <script type="importmap">
  {{
    "imports": {{
      "react":             "/vendor/react.mjs",
      "react-dom/client":  "/vendor/react-dom-client.mjs",
      "react/jsx-runtime": "/vendor/react-jsx-runtime.mjs"
    }}
  }}
  </script>
  <style>
    * {{ box-sizing: border-box; margin: 0; padding: 0; }}
    body {{ font-family: system-ui, sans-serif; padding: 24px; background: #f9f9f9; }}
    #root {{ max-width: 960px; margin: 0 auto; }}
    #state-loading {{ color: #666; padding: 40px; text-align: center; }}
    #state-error {{
      color: #c00; background: #fff0f0; border: 1px solid #f99;
      border-radius: 6px; padding: 16px; margin: 16px 0;
    }}
    #state-error pre {{ font-size: 12px; margin-top: 8px; white-space: pre-wrap; }}
  </style>
</head>
<body>
  <div id="root">
    <div id="state-loading">Loading...</div>
  </div>

  <script type="module">
    import * as ReactDOM from "react-dom/client";
    import * as React from "react";

    const COMPONENT_NAME = "{component_name}";
    const INPUT_ID = "{input_id}";

    function showError(title, detail) {{
      const root = document.getElementById("root");
      root.innerHTML = `
        <div id="state-error">
          <strong>${{title}}</strong>
          <pre>${{detail}}</pre>
        </div>`;
    }}

    async function main() {{
      // 1. Load the component bundle
      let ComponentModule;
      try {{
        ComponentModule = await import(`/api/components/${{COMPONENT_NAME}}.js`);
      }} catch (err) {{
        showError("Failed to load component", err.message);
        return;
      }}

      const Component = ComponentModule.default;
      if (!Component) {{
        showError("Invalid component", `${{COMPONENT_NAME}}.js has no default export.`);
        return;
      }}

      // 2. Fetch the input object
      let inputData;
      try {{
        const res = await fetch(`/api/inputs/${{INPUT_ID}}`);
        if (!res.ok) throw new Error(`HTTP ${{res.status}}: ${{await res.text()}}`);
        inputData = await res.json();
      }} catch (err) {{
        showError(`Failed to load input object (id: ${{INPUT_ID}})`, err.message);
        return;
      }}

      // 3. Render — inputData keys are spread directly as component props
      const rootEl = document.getElementById("root");
      const reactRoot = ReactDOM.createRoot(rootEl);

      class ErrorBoundary extends React.Component {{
        constructor(props) {{ super(props); this.state = {{ error: null }}; }}
        static getDerivedStateFromError(err) {{ return {{ error: err }}; }}
        render() {{
          if (this.state.error) {{
            return React.createElement("div", {{ id: "state-error" }},
              React.createElement("strong", null, "Component render error"),
              React.createElement("pre", null, this.state.error.message)
            );
          }}
          return this.props.children;
        }}
      }}

      reactRoot.render(
        React.createElement(ErrorBoundary, null,
          React.createElement(Component, inputData)
        )
      );
    }}

    main();
  </script>
</body>
</html>"""
```

### Key Design Choices in the Shell

- **No build step.** The shell is a plain HTML string with a `<script type="module">`. No Vite, no bundler.
- **Import map for React.** User component bundles mark `react` etc. as external; the import map resolves them to vendored ESM files, guaranteeing a single React instance.
- **Input data spread as props.** `React.createElement(Component, inputData)` spreads the fetched JSON directly as props. The `InputProps` interface in the `.tsx` file is therefore the exact shape of the stored input object — the contract is end-to-end with no intermediate mapping.
- **Error handling at every async step.** Network failure, missing default export, and render errors all show human-readable in-page messages.
- **No SPA, no router.** Each `/labeling/` URL is a full page load. Simple and debuggable.

```bash
pip install yourtool
```

```bash
yourtool scaffold trace_comparison
```

```bash
python -c "import yourtool; yourtool.serve()"
```

Output:
```
[yourtool] Scanning /home/user/project/labeling for labeling components...
[yourtool] Downloading esbuild 0.20.2 for darwin/arm64... (~7MB, one-time)
[yourtool] ✓ classification_check
[yourtool] ✓ llm_output_review
[yourtool] ✓ trace_comparison

```

### Accessing a labeling page

```
http://localhost:8080/labeling/trace_comparison?id=eval_run_abc123_item_7
```

What happens:
1. FastAPI looks up `trace_comparison` in the registry → found
2. Returns the HTML shell with `component_name` and `input_id` baked in as JS constants
3. Browser executes `<script type="module">`
4. Shell imports `/api/components/trace_comparison.js` (the esbuild bundle)
5. Shell fetches `/api/inputs/eval_run_abc123_item_7` → receives JSON object
6. Shell calls `ReactDOM.createRoot(root).render(<TraceComparison {...inputData} />)`
7. User sees their custom labeling UI populated with real data

## 12. Testing Requirements

### Unit Tests

- `test_esbuild.py`: Mock `urllib.request.urlopen`. Assert the correct tarball URL is constructed per platform. Assert the binary is written to the correct cache path. Assert the env var override short-circuits the download.
- `test_bundler.py`: With a real or mocked esbuild binary. Valid `.tsx` → produces a `.js` file starting with ESM output. Invalid TSX syntax → raises `ComponentBundleError` with esbuild stderr in the message.
- `test_scanner.py`: Temp dir with two valid `.tsx` files and one invalid one. Assert two entries in registry, one warning printed, no exception raised.
- `test_registry.py`: Basic set/get/list operations.

### Integration Test

Create a minimal `.tsx` file in a temp dir, call `scan_and_register()`, assert the bundle file exists and is non-empty.
